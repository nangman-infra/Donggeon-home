Design and implement a premium personal website for an AI Product Engineer.

IMPORTANT:

This is NOT a student portfolio.
This is NOT a resume website.
This is NOT a dashboard.

The final result should feel closer to a funded AI startup website than a traditional developer portfolio.

The visual inspiration should come from:

- Dalpha
- Linear
- Stripe
- Vercel
- Anthropic
- Raycast

Do not copy these websites directly.

Instead, capture their design philosophy.

==================================================
OVERALL GOAL
==================================================

When someone visits this website, they should think:

"This person builds real AI products."

before they think:

"This person is looking for a job."

The website should communicate:

- engineering ability
- product thinking
- AI expertise
- execution capability
- production experience

The website must tell a story.

It should not simply list projects.

==================================================
VISUAL STYLE
==================================================

Premium AI startup aesthetic.

Modern.

Editorial.

Minimal.

Sophisticated.

Confident.

Large typography.

Strong hierarchy.

Massive whitespace.

Dark mode first.

Use visual restraint.

Avoid visual clutter.

Avoid excessive colors.

Avoid generic portfolio aesthetics.

Avoid student project showcase aesthetics.

Avoid resume aesthetics.

==================================================
STRICTLY AVOID
==================================================

Do NOT create:

- sidebar navigation
- profile card layouts
- skill progress bars
- technology badges
- generic project cards
- boxed sections everywhere
- portfolio template layouts
- GitHub profile style pages
- timeline resumes
- card grids for everything

If the design resembles a portfolio template,
reject it and redesign from scratch.

==================================================
ANIMATION SYSTEM
==================================================

Motion is a first-class design element.

Animations must feel premium.

Animations must support storytelling.

Do NOT use generic animations.

Avoid:

- simple fade in
- slide up on scroll
- hover scale everywhere
- repetitive animation patterns

Use:

- scroll-linked animations
- staggered text reveals
- blur-to-focus transitions
- progressive content transformations
- subtle parallax
- pinned storytelling sections
- cinematic section transitions

Motion should feel similar to:

- Dalpha
- Linear
- Stripe

==================================================
HERO ANIMATION
==================================================

Headline appears line by line.

Starts slightly blurred.

Sharpens into focus.

Background grid subtly moves.

Mouse movement creates a subtle parallax effect.

Animation should feel calm and premium.

==================================================

==================================================

==================================================
MICRO INTERACTIONS
==================================================

Buttons:

Subtle magnetic effect.

Smooth hover transitions.

Navigation:

Glassmorphism used sparingly.

Smooth section transitions.

Scrolling:

Highly polished.

No abrupt changes.

==================================================
TECHNICAL EXPECTATIONS
==================================================

Use modern frontend practices.

Framer Motion preferred.

GSAP ScrollTrigger can be used where appropriate.

Animations should remain performant.

Prioritize smoothness.

Avoid animation overload.

==================================================
FINAL DESIGN CHECK
==================================================

Before finalizing, verify:

1. Does it feel like a startup website rather than a portfolio?

2. Does it communicate AI product engineering?

3. Does it avoid portfolio-template aesthetics?

4. Does scrolling feel like storytelling?

5. Does the design feel premium enough to sit alongside Dalpha, Linear, or Vercel?

If any answer is no,
redesign before finalizing.

The current design looks too much like a generic computer science student portfolio.

Completely rethink the information architecture.

Prioritize branding, storytelling, motion design, and product presentation over personal information.

The website should feel like the homepage of an AI startup founded by the engineer, not a resume website.

==================================================
LANGUAGE & CONTENT RULES
==================================================

The website audience is Korean recruiters, engineers, and hiring managers.

All primary content should be written in Korean.

Examples:

- Headlines
- Section titles
- Descriptions
- Project explanations
- Engineering philosophy
- About section

should all be Korean by default.

English may be used only when it improves branding or readability.

Examples:

- AI Product Engineer
- Browser Agent
- Enterprise RAG
- Orbit

are acceptable.

However, avoid fully English paragraphs.

The website should feel natural for a Korean audience.
